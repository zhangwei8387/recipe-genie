# 用户故事（User Stories）

## US-001 — 匿名访谈（2025-12-27）
- 描述：作为一个每周做饭 3–4 次、单次约 1 小时并偏好炒菜的用户，我希望系统能推荐与我常做菜式相匹配的可执行菜谱，并能清楚显示预估时间和步骤复杂度。
- 验收标准（初稿）：
  - 推荐结果中能标注预计烹饪时间，优先展示与“炒菜”相关的菜谱。
  - 对于用户设定的时间（例如 1 小时）能过滤出可执行的候选食谱（Top5）。
  - 在 20–30 分钟的时间约束下，优先展示步骤简洁的食谱（准备/步骤少），且能在 2–3 步内开始烹饪。
- 来源：`user_interviews/2025-12-27_anon_1.md`

## US-002 — 匿名访谈（2025-12-27）
- 描述：作为一个在看到冰箱食材时倾向随便尝试的用户，我希望系统能提供“灵感”或“随意试试”模式，给出简单、低门槛的建议与变体，并在需要时提供快速过滤选项（按时间或口味）。
- 验收标准（初稿）：
  - 提供“灵感”或“随机推荐”模式；在该模式下，推荐应突出简易性（较少预备步骤）并能在 2 步内开始烹饪。
  - 优先满足“节省时间”诉求：建议应能在 2–3 分钟内产生可执行方案，步骤简洁、准备材料最少。
  - 用户可以从推荐中快速切换到按时间或营养过滤的视图。
- 来源：`user_interviews/2025-12-27_anon_1.md`
